Las bases de datos transaccionales son almacenes de filas, es decir, que se almacen de manera horizontal en lugar de columnas(vertical). Estas estas optimizadas para los sitemas de producción en ejecución, destacan por leer y escribir filas de datos concretas de manera muy rápida sin debiitar la integridad de los datos.

Son la solución ideal para consultar toda la informaciónde un cliente en la tabla de usuarios, sin embargo, no son tan eficaces cuando intentas contar ciertas cosas en especifico, por ejemplo, los clientesde un código postal especifico.

El inconveniente es que estas bases de datos están diseñadas para procesar transacciones, no análisis.

  • Ventajas
    • Asegurar la integridad de datos. Las bases de datos transaccionales están diseñadas para cumplor los requisitos de ACID. Estas son fundamentales para las transacciones empresariales en las que se necesita un alto nivel de integridad de datos.
    • Latencia baja. Estan diseñadas para ejecutar sistemas de producción, destacan en las operaciones que tienen que completarse en milisegundos. Si haces analíticas en una réplica transaccionalde la base de datos de producción, es probable que la réplica esté prácticamente sincronizada con la base de datos principal, es decir, que tenga una latencia inferior a un segundo.
    • Monitorizar sistemas operativos. Trabajar con datos de bases de datos transaccionales para proporcionar una captura dela operacion en tiempo real es un caso de uso perfecto para las bases de datos transaccionales.
  • Transacción ACID. Es un conjunto de propiedades que describen cómo se diseñan las bases de datos transaccionales para conservar la integridad de las operaciones de escritura en la base de datos.
    • (A)tomicidad. Aunque falle una parte de la transacción, falla toda. De esta forma, cada transacción tiene que completarse al 100% para que se guarde correctamente.
    • (C)oherencia. Se escribe la transacción en la base de datos o se anula la transacción.
    • A(I)slamiento. Las transacciones que aún no se hayan completado no se pueden usar. Otras transacciones pueden modificarlas.
    • (D)urabilidad. Una vez que se haya escrito una transacción en la base de datos, permanecerá en ella aunque produzca un fallo.
